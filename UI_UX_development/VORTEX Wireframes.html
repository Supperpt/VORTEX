Approach B · Terminal

Polished TUI stateful terminal dashboard

Stays CLI-native — no GL, no window-server fragility — but gives the shell a real dashboard. A persistent status table, the pipeline drawn as a state machine, the clip-sac stats rendered with an ASCII bulge heatmap, and a command line with ghosted suggestions. This is the safest home given the README's X11/Wayland history.

Approach C · Desktop

Focus Stage calm, one-thing-at-a-time

The clinical, low-anxiety option. A full-bleed canvas, a single floating workflow stepper up top (progress, not a menu), and just the controls the current step needs floating in — big targets, advanced params tucked behind a disclosure. Shown at the seed picking moment; calmer for the clinician side of a mixed audience.

Approach D · Desktop

Pipeline Graph spatial node canvas

The whole pipeline as a flow of connected nodes — you see the entire state and data dependencies at once. Arrows encode the optimal order (the external-edit hop is a dashed detour out and back); click any node to expand its inspector with params + a preview thumbnail. Run nodes in any order — the graph just shows what's stale. Shown with the Segment node selected.
